# Происхождение и независимая реализация

Nayti создаётся как самостоятельная greenfield-кодовая база с новой Git history. Репозиторий не является fork, template, subtree, импортом или очищенной веткой SmartScan либо другого приложения.

## Допустимые источники

При проектировании используются публично наблюдаемое поведение Android, официальная документация платформы и библиотек, научные публикации, model cards, открытые стандарты и независимо сформулированные требования и тесты.

Общие идеи — локальный индекс, OCR, foreground service, MediaStore reconciliation, полнотекстовый и векторный поиск — не определяют конкретную реализацию. Код, package names, Gradle-файлы, manifests, database schemas, строки, тесты, UI, изображения и store assets других приложений не переносятся.

## Сторонние компоненты

Каждая runtime-зависимость и модель должна иметь зафиксированные источник, revision, лицензию и, для binary artifacts, криптографический hash. Лицензия Apache-2.0 относится к собственному коду Nayti и не заменяет лицензии моделей или библиотек.

Model weights, tokenizer artifacts и собранные model packs не публикуются в Git history. Перед release для них формируются отдельные notices, checksums и provenance metadata.

## Проверка изменений

Изменения описываются через реализуемый контракт Nayti, а не через соответствие файлу стороннего приложения. Перед release выполняются dependency/license audit, проверка binary artifacts и выборочная provenance review.

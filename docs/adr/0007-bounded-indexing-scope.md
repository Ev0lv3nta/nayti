# ADR-0007: ограниченная область первичной индексации

Статус: принято 19 июля 2026 года.

## Контекст

Полная локальная обработка большой фототеки требует реального времени устройства, места и энергии. Для первой проверки качества разумнее закончить репрезентативную недавнюю выборку, оценить поиск и только затем расширить охват. Жёстко заданный период не подходит: размер и возраст галерей различаются, а диагностический прогон должен быть воспроизводимым.

Ограничение индексации не является ограничением Android photo access. Каталог обязан продолжать учитывать всю доступную MediaStore-библиотеку, чтобы корректно обрабатывать разрешения, удаления и последующее расширение периода.

## Решение

- Пользователь выбирает всю медиатеку, один из быстрых интервалов 1/3/6/12 месяцев или произвольную начальную дату. Интервал всегда продолжается до настоящего времени.
- При выборе сохраняется точная нижняя временная граница и новая revision. Граница не сдвигается ежедневно: один и тот же многодневный прогон остаётся воспроизводимым, а новые фотографии автоматически попадают в него.
- Eligibility использует `DATE_TAKEN`, а при его отсутствии — `DATE_MODIFIED`. Записи без обеих дат остаются в каталоге, явно учитываются в сводке и не входят в ограниченный период. Режим всей медиатеки включает их.
- Durable operation содержит только assets текущей revision области. Claim обязательно проверяет membership в operation, поэтому незавершённая работа от более широкого старого прогона не может случайно выполниться.
- OCR, semantic, visual и pHash publications при сужении не удаляются. При расширении готовые совместимые publications переиспользуются, а планируются только отсутствующие или устаревшие результаты.
- Активная область применяется ко всем пользовательским выдачам: unified search, similar, duplicates, viewer evidence и search facets. Смена revision во время unified query считается изменением snapshot consistency.
- Период нельзя менять во время активного execution window. После паузы смена выполняется атомарно, получает новый operation identity и не повреждает завершённую работу.
- Время прогона считается как сумма активных execution windows, без времени пауз и ожидания системных условий. Прогноз полной библиотеки появляется только после завершения всех каналов выбранной выборки и линейно масштабирует активное время по числу assets.

## Последствия

Первый device-прогон можно ограничить недавними фотографиями, получить законченную выдачу и сопоставимую оценку до многочасовой полной обработки. Расширение периода не требует перестройки уже готовой части, но active vector snapshot всё равно публикуется по существующему атомарному протоколу.

Прогноз полной медиатеки остаётся ориентиром: старые фотографии могут отличаться по разрешению, объёму текста и декодированию, а thermal throttling меняет throughput. Поэтому UI и диагностика называют его предварительным, а фактическое решение о полном прогоне принимается по device evidence.

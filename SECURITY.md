# Политика безопасности

## Поддерживаемые версии

Проект ещё не выпускал публичную alpha-версию. До появления первого release security fixes применяются только к текущей ветке `main`.

## Как сообщить об уязвимости

Используйте private vulnerability reporting в разделе Security репозитория. Не создавайте публичный Issue с эксплуатационными деталями или пользовательскими данными.

В сообщении укажите затронутый commit или версию, условия воспроизведения, возможное влияние и минимальный proof of concept. Не отправляйте реальные фотографии, OCR-текст, поисковые запросы, ключи, токены или model packs.

Получение сообщения будет подтверждено после первичного разбора. Срок исправления зависит от воспроизводимости и влияния; детали публикуются после появления безопасного обновления.

## Граница проекта

Nayti не должен иметь разрешения `INTERNET` и обрабатывает медиатеку локально. Уязвимостями считаются, в частности, утечки через logs/diagnostics, обход проверки model pack, чтение отозванных MediaStore-данных, небезопасные exported components и повреждение active index при восстановлении.
